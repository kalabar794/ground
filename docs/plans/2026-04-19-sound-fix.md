# iPhone Safari Sound Fix Plan

**Goal:** Make the built-in soundscapes reliably audible and usable on iPhone Safari.

**Root-cause hypothesis:** The audio graph starts, but the default mixes are tuned too low and too bass-heavy for phone speakers. The default selected sound is `rain`, which uses heavily filtered brown noise at a low cutoff, so users can interpret it as broken even when playback state changes correctly.

## Tasks

1. Reproduce the current sound behavior on the live page and inspect the existing audio graph.
2. Keep the current tap-gated Web Audio startup path.
3. Retune the generated soundscapes toward more audible midrange output on phone speakers:
   - use brighter source/filter settings for `rain` and `ocean`
   - raise gain conservatively
   - keep `forest` distinct
4. Verify:
   - no deprecated `ScriptProcessorNode`
   - play button toggles `Play`/`Stop`
   - sound state label updates
   - no JS errors after playback starts
5. Push to GitHub Pages and verify the live URL serves the patched code.
